# AgencyOS 密钥安全配置指南

## 概述
本文档介绍如何安全地配置和管理AgencyOS的加密密钥，以确保系统的安全性。

## 密钥要求

### 长度要求
- JWT加密密钥应至少使用 **32字节（256位）**
- 推荐使用 64 字节（512位）以获得更高安全性
- 使用安全的随机数生成器生成密钥

### 生成安全密钥
```bash
# 方法1: 使用Python生成
python3 generate_key.py

# 方法2: 直接使用Python命令
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

## 安全配置步骤

### 1. 创建环境配置文件
```bash
# 复制示例配置文件
cp .env.example .env
```

### 2. 生成并配置密钥
```bash
# 生成新的密钥
python3 generate_key.py --length 32

# 将输出的密钥替换 .env 文件中的 ENCRYPTION_KEY 值
```

### 3. 设置文件权限
```bash
# 设置 .env 文件权限，仅允许当前用户读写
chmod 600 .env
```

## 安全最佳实践

### 环境变量管理
- 不要在代码中硬编码密钥
- 使用环境变量管理敏感信息
- 将 `.env` 文件添加到 `.gitignore` 中

### 部署注意事项
- 生产环境使用专门的密钥管理系统（如HashiCorp Vault、AWS Secrets Manager）
- 定期轮换密钥
- 监控密钥使用情况

## 验证配置

启动服务前，可以通过以下方式验证密钥配置：

```bash
# 检查环境变量是否正确设置
echo $ENCRYPTION_KEY
```

## 注意事项

1. **不要提交密钥到版本控制**：确保 `.env` 文件在 `.gitignore` 中
2. **定期更换密钥**：建议每3-6个月更换一次密钥
3. **备份密钥**：更换密钥时，需要重新登录所有活跃用户
4. **权限控制**：确保只有授权进程可以访问密钥

## 故障排除

### 如果忘记更新密钥
- 所有现有的JWT令牌将失效
- 用户需要重新登录
- 重新生成密钥并更新配置

### 如果密钥泄漏
- 立即更换密钥
- 通知用户需要重新登录
- 检查系统日志以检测未授权访问
